# Project Context: AI Project Configuration

## 🎯 Overview
This repository is the central "Source of Truth" for your local AI development environment. It manages rules, workflows, and initialization scripts to enforce a standardized and isolated development process.

## 🏗️ Architecture: Hybrid KVM-Worker
The project enforces a strict separation between planning and execution:

- **Planner Plane (Host)**: Handled by Gemini (Antigravity). Responsible for architecture, product strategy, and documentation. Uses `core-planner/` rules.
- **Worker Plane (KVM)**: Handled by Pi (pi.dev) inside an Incus VM. (Note: Pi refers to the pi.dev coding agent, not Raspberry Pi hardware). Responsible for implementation, testing, and debugging. Uses `core-worker/` rules.

## 📂 Key Components
- `core-planner/`: Instructions for the host-side agent.
- `core-worker/`: Instructions and guest-init scripts for the VM-side agent.
- `scripts/`: Orchestration scripts (e.g., `init-agents.sh`, `worker-bridge.sh`).
- `docs/`: Architectural documentation and project backlog.

## 🛠️ Recent Technical Shifts
- **Incus Integration**: Fully migrated to Incus VMs for superior isolation and performance.
- **Worker Pivot**: Switched from Opencode to Pi (pi.dev) to resolve JSON decoding issues and improve autonomous task execution reliability.
- **Role-Based MCP Filtering**: Implemented role-specific `config.jsonc` files to restrict `jcodemunch-mcp` tools for Planner and Worker roles.
- **Pi Configuration**: Implemented a direct-mount strategy for `~/.pi/agent` to manage agent extensions, rules, and workflows.

## 🚦 Current Status
- **Core Architecture**: Stable (Pi + Incus).
- **Backlog**: Restructuring Pi assets for simplicity and fixing MCP visibility.
- **Next Steps**: Verifying autonomous loop reliability with Pi.
