# ADR-001: Standardize Core Worker Asset Resolution

* **Status:** Proposed
* **Date:** 2026-04-27 (Initial)
* **Corpus/Module:** ai-project-configuration: scripts/worker-bridge.sh

## 📋 Context
The current `worker-bridge.sh` script relies on a calculated `REPO_ROOT` based on the script's own location (`dirname $(realpath $(dirname "$0"))`). This has proven fragile when the script is invoked from different locations or via symlinks, leading to errors like `Missing source path "/home/mike/Projects/core-worker"`. The user's specific project layout (`~/Project/{identity}/{project-name}`) exposes this fragility.

## 🎯 Decision
Standardize the resolution of `core-worker` assets to use the `${HOME}/.agents/core-worker` directory as the primary reference point. This directory is already established by `init-agents.sh` as a stable link to the repo's `core-worker/` folder.

1.  Modify `worker-bridge.sh` to prioritize `~/.agents/core-worker` for the `core-worker` mount source.
2.  Fall back to relative path calculation only if the standardized path is missing.
3.  Ensure all references to templates and guest scripts use this standardized path.
4.  Standardize all references to `jcodemunch-mcp` and `jdocmunch-mcp` to `jcodemunch-mcp` and `jdocmunch-mcp` across all scripts, rules, and documentation to align with the actual binary names and avoid developer confusion.

## ⚖️ Consequences
- **Positive:** Improved robustness across different project layouts and invocation methods. Consistent with the established `init-agents.sh` setup.
- **Negative:** Adds a dependency on `init-agents.sh` having been run previously (which is already a prerequisite for the system).

## 🔄 Alternatives Considered
- Fixing the `REPO_ROOT` logic: Rejected because even "perfect" logic can be foiled by complex symlinking or environment differences.
- Using environment variables: Rejected to avoid polluting the user's shell environment and maintaining "stateless" script execution where possible.
