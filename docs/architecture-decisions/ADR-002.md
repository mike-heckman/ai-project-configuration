# ADR-002: Switching Worker Agent to Opencode

## Status
Proposed

## Context
The current worker agent implementation uses `Claude Code` (v0.x) running against local models (e.g., Gemma-4 via LM Studio). We have encountered a "brick wall" with JSON decoding issues where the agent fails to correctly parse tool calls or outputs from the local model, leading to system failure. 

To resolve this, we need a more robust agent that can handle local model outputs more reliably. `Opencode` has been identified as a suitable replacement that integrates better with our on-prem LLM setup (Ollama).

## Decision
We will switch the default worker agent from `Claude Code` to `Opencode` within the Incus KVM environment. 

Key changes include:
1.  Updating `scripts/worker-bridge.sh` to support `Opencode` configuration.
2.  Updating `core-worker/kvm/guest-init.sh` to install and configure `Opencode` instead of `Claude Code`.
3.  Utilizing `core-worker/kvm/opencode.json` for agent configuration.
4.  Updating environment templates to reflect the new agent name.

## Consequences
-   Improved stability and JSON decoding reliability when using local models.
-   The worker environment will now rely on `Opencode` as the primary execution engine.
-   Existing rules, roles, and workflows will be preserved and passed to `Opencode`.
-   Configuration for `Opencode` will be managed via `opencode.json`.
