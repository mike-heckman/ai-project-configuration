# ADR-003: Switching Worker Agent to Pi

## Status
Proposed

## Context
Following the transition to `Opencode` in ADR-002, we attempted to stabilize the autonomous worker agent for multi-step agentic loops. Despite extensive refinements to the persona, rules, and skills, `Opencode` failed to maintain a reliable autonomous loop when working with local models (e.g., Gemma-4). The agent frequently stalled or failed to correctly process tool outputs in a way that supported continuous execution without manual intervention.

To achieve a truly "never-stop" autonomous workflow, we need an agent that provides deeper hooks into its execution state and more robust handling of structured output. `Pi` (@mariozechner/pi-coding-agent) is a minimal, highly extensible terminal coding harness that specifically addresses these needs through:
1.  **RPC Mode (`--mode rpc`)**: Allows for granular, process-level control and monitoring of the agent's actions.
2.  **JSON Event Stream (`--mode json`)**: Provides structured events for all actions, making it easier to build reliable orchestration layers.
3.  **Local Model First-Class Support**: Simplified configuration for Ollama/LM Studio via `models.json`.
4.  **Extensibility**: Native support for TypeScript extensions, skills, and prompt templates.

## Decision
We will migrate the worker agent from `Opencode` to `Pi` within the Incus KVM environment.

Key implementation steps include:
1.  Updating `core-worker/kvm/guest-init.sh` to install `pi` globally via npm.
2.  Creating a configuration structure for `pi` in `core-worker/kvm/pi/` (replacing `opencode/`).
3.  Configuring `~/.pi/agent/models.json` to route requests to the local Ollama/LM Studio endpoints.
4.  Utilizing `AGENTS.md` for project-specific instructions and persona hardening.
5.  Integrating `pi --mode rpc` or `pi --mode json` into our orchestration logic for improved reliability.

## Consequences
-   **Improved Reliability**: The switch to `Pi` provides a more stable foundation for long-running autonomous tasks.
-   **Structured Interaction**: The availability of RPC and JSON modes enables more robust monitoring and error recovery.
-   **Configuration Change**: Agent configuration shifts from `opencode.json` to a set of files in `~/.pi/agent/` (`models.json`, `settings.json`, `AGENTS.md`).
-   **Prerequisite Change**: The worker environment now requires `node` and `npm` for the `pi` agent, moving away from the `uv`-centric setup used for `Opencode`.
