# 🏁 Workflow: /ready (Definition of Done)

**MANDATORY:** You are prohibited from declaring a task finished until this loop returns a Success state.

## 1. Automated Verification Suite
- **Execute `/lint`:** Must return Exit 0. 
- **Execute `/test`:** Must return Exit 0.
- **Stop Condition:** If either fails, analyze `logs/`, apply fixes, and **restart** from Step 1.

## 2. Coverage & Regression Gate
- **Read:** `./coverage.md` (generated by `/test`).
- **Constraint:** New logic MUST have test coverage. If coverage decreased or new lines are "Missing," you are **PROHIBITED** from finishing. 
- **Action:** Add missing tests and restart from Step 1.

## 3. Documentation Sync
- **Execute `/update-docs`:** Ensure all manual and auto-generated docs are current.

## 4. MCP Index Integrity (Source of Truth)
- **Code:** Call `jCodeMunch.index_file` for every modified or new `.py` file.
- **Docs:** Call `jDocMunch.index_documentation` if `docs/` or public APIs changed.
- **Verify:** Use `jCodeMunch.get_file_outline` on a primary modified file to confirm the index reflects the new state.

## 5. Final Handover Protocol
Provide the following status block as the final output of the task:

> ### ✅ Status: **Release Ready**
> - **Verification:** All workflows passed (Exit 0).
> - **Sync:** Re-indexed [List Files] in jCodeMunch/jDocMunch.
> - **Git Policy:** No Git actions (add/commit/push) performed.
