description: Task Completion Protocol (The "Definition of Done")
---
**MANDATORY:** Before declaring a task finished or notifying the user, you must successfully complete this loop.

### 1. Verification Execution
- Invoke the **`/lint`** workflow. **Requirement:** Must return Exit 0.
- Invoke the **`/test`** workflow. **Requirement:** Must return Exit 0. If the command failed, do not proceed to the next step of the calling workflow.
- *Failure Logic:* If either workflow fails, analyze the output, apply a fix, and restart from Step 1.

### 2. Coverage Validation
- **Read** the `coverage.md` file generated by the `/test` workflow.
- **Constraint:** All new logic must have test coverage. If coverage has decreased or new lines are marked as "Missing," you are prohibited from finishing. Add the missing tests and restart the protocol from Step 1.

### 3. Update documentation
- Invoke the /update-docs workflow

### 4. MCP Synchronization (Source of Truth)
- **Code Index:** Call `jCodeMunch.index_file` for every modified or new `.py` file.
- **Docs Index:** If `docs/` were modified or public APIs changed, call `jDocMunch.index_documentation`.
- **Symbol Check:** Briefly use `jCodeMunch.get_file_outline` on a modified file to confirm the index reflects your changes.

### 5. Final Handover Statement
Explicitly state the following in your final response:
> Status: **Release Ready**. 
> Verification: All scripts passed (Exit 0). 
> Sync: Re-indexed [list files here] in jCodeMunch/jDocMunch. 
> Git: No Git actions (add/commit) performed.
